Como expert em GitHub, selecionei as 8 perguntas fundamentais que estruturam um README de alta qualidade (seguindo as melhores práticas de documentação).

Aqui estão as perguntas. Você pode copiar o conteúdo abaixo e salvar no seu arquivo `projeto.txt`.

### 8 Perguntas Essenciais para o seu README

1.  **O que é este projeto?** (Nome, descrição clara e o objetivo principal).
2.  **Por que este projeto existe?** (Qual problema ele resolve ou qual necessidade ele atende).
3.  **Quais são as principais funcionalidades (features)?** (Uma lista do que o software é capaz de fazer).
4.  **Como instalo e configuro o ambiente?** (Passo a passo dos pré-requisitos e comandos de instalação).
5.  **Como utilizo o projeto?** (Exemplos práticos de uso, prints ou trechos de código).
6.  **Como outros desenvolvedores podem contribuir?** (Regras para Pull Requests, guia de estilo e reporte de bugs).
7.  **Quem são os autores/mantenedores?** (Nomes e links para contato ou perfis).
8.  **Sob qual licença este projeto está distribuído?** (MIT, Apache, GPL, etc).
